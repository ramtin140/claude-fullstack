# FIFA Soul — پلتفرم تورنمنت‌های فوتبال دیجیتال

یک اپلیکیشن فول‌استک برای برگزاری تورنمنت‌ها و مسابقات آنلاین فیفا، با طراحی الهام‌گرفته از قالب گرافیکی FIFA Soul (تم بنفش/طلایی، راست‌به‌چپ، فونت Vazirmatn).

## معماری

```
.
├── backend/   # Node.js + Express + SQLite (REST API, JWT auth)
└── frontend/  # React + Vite (RTL, Persian UI)
```

- **بک‌اند**: Express روی SQLite (`better-sqlite3`) با احراز هویت JWT، نقش‌های `member`/`admin`، و مسیرهای REST برای تورنمنت‌ها، مسابقات، اخبار، آمار، کاربران و خبرنامه.
- **فرانت‌اند**: React 18 + React Router با فونت متغیر Vazirmatn (سبک‌ترین فونت فارسی با پوشش کامل وزن‌ها در یک فایل) و آیکون‌های سبک `lucide-react`.
- **پنل مدیریت**: مسیر `/admin` (فقط برای نقش admin) برای مدیریت کامل تورنمنت‌ها، مسابقات، اخبار و کاربران.

## اجرا در محیط توسعه

### ۱. بک‌اند

```bash
cd backend
npm install
cp .env.example .env   # در صورت نیاز مقادیر را تغییر دهید
npm run dev             # http://localhost:4000
```

اولین اجرا به‌صورت خودکار دیتابیس SQLite را در `backend/data/fifasoul.db` می‌سازد و با داده‌های نمونه seed می‌کند.

کاربر ادمین پیش‌فرض: `admin@fifasoul.test` / `admin123`

### ۲. فرانت‌اند

```bash
cd frontend
npm install
npm run dev              # http://localhost:5173
```

درخواست‌های `/api/*` در حالت توسعه به‌صورت خودکار به بک‌اند (پورت ۴۰۰۰) پروکسی می‌شوند.

## دیپلوی روی Railway (دو سرویس جدا)

بک‌اند و فرانت‌اند به‌صورت دو سرویس مستقل روی Railway دیپلوی می‌شوند و از طریق HTTP با هم صحبت می‌کنند (نه سرو کردن فایل‌های استاتیک از یک سرویس).

### ۱. سرویس بک‌اند

- **Root Directory**: `backend`
- **Build Command**: `npm install`
- **Start Command**: `npm start`
- **Volume**: یک Volume پایدار روی مسیر `/data` متصل کنید (برای فایل SQLite)

متغیرهای محیطی سرویس بک‌اند:

| متغیر | مقدار |
|---|---|
| `JWT_SECRET` | یک رشته تصادفی و امن |
| `FRONTEND_URL` | آدرس کامل سرویس فرانت‌اند روی Railway، مثلاً `https://fifasoul-frontend.up.railway.app` (برای مجاز کردن CORS) |
| `PORT` | نیازی به تنظیم دستی نیست؛ Railway خودش مقداردهی می‌کند |
| `DATABASE_PATH` | اختیاری. اگر ست نشود و روی Railway اجرا شوید (با تشخیص خودکار متغیرهای `RAILWAY_*`)، مسیر پیش‌فرض `<مسیر mount شده ولوم>/fifasoul.db` است (یا `/data/fifasoul.db` در نبود آن متغیر). برای اطمینان کامل می‌توانید صریحاً `DATABASE_PATH=/data/fifasoul.db` را هم ست کنید |

### ۲. سرویس فرانت‌اند

- **Root Directory**: `frontend`
- **Build Command**: `npm install && npm run build`
- **Start Command**: یک static server برای پوشه `dist`، مثلاً `npx serve -s dist -l $PORT` (یا هر Railway static-site preset مشابه)

متغیر محیطی سرویس فرانت‌اند:

| متغیر | مقدار |
|---|---|
| `VITE_API_URL` | آدرس کامل سرویس بک‌اند روی Railway، مثلاً `https://fifasoul-backend.up.railway.app` (چون این مقدار در زمان build داخل باندل جاوااسکریپت جاسازی می‌شود، هر بار که آدرس بک‌اند عوض شد باید فرانت‌اند را دوباره build/دیپلوی کنید) |

اگر `VITE_API_URL` ست نشود، فرانت‌اند از مسیر نسبی `/api` استفاده می‌کند که فقط در حالت توسعه محلی (با پروکسی Vite) معنی دارد؛ برای دیپلوی این متغیر الزامی است.

### بعد از دیپلوی

با اولین اجرای بک‌اند، اسکیمای دیتابیس و داده‌های نمونه به‌صورت خودکار روی Volume ساخته می‌شوند. چون این دو سرویس روی دامنه‌های جدا هستند، بک‌اند باید `FRONTEND_URL` را برای CORS بشناسد و فرانت‌اند باید `VITE_API_URL` را برای پیدا کردن API بشناسد — هر دو باید دقیقاً با آدرس نهایی Railway هر سرویس هم‌خوانی داشته باشند.

## امکانات

- ثبت‌نام / ورود کاربران با رمزنگاری bcrypt و توکن JWT
- مدیریت تورنمنت‌ها (لیگ/کاپ) و ثبت‌نام کاربران در آن‌ها
- مدیریت مسابقات با وضعیت زنده/در انتظار/پایان‌یافته
- بخش اخبار با دسته‌بندی (بازی‌های فعال، جدیدترین‌ها، گیک‌ها، آموزش‌ها)
- آمار خانه (رو در رو، پلی‌آف، لیگ، کاپ) و جدول اعضای برتر — همگی از دیتابیس واقعی محاسبه می‌شوند
- عضویت در خبرنامه با شماره موبایل
- پنل مدیریت کامل با CRUD برای تورنمنت‌ها، مسابقات، اخبار و کاربران

## نکته درباره مستندات

ریپازیتوری در ابتدا فقط یک قالب خالی Streamlit بود و مستنداتی به‌جز تصویر طراحی صفحه اصلی همراه درخواست ارسال نشده بود؛ بنابراین معماری، مدل داده و صفحات بر اساس تحلیل تصویر و بهترین شیوه‌های متداول برای این نوع پلتفرم طراحی شدند.
