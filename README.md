# FIFA Soul — پلتفرم تورنمنت‌های فوتبال دیجیتال

یک اپلیکیشن فول‌استک برای برگزاری تورنمنت‌ها و مسابقات آنلاین فیفا، با طراحی الهام‌گرفته از قالب گرافیکی FIFA Soul (تم بنفش/طلایی، راست‌به‌چپ، فونت Vazirmatn).

## معماری

```
.
├── backend/   # Node.js + Express + SQLite (REST API, JWT auth)
└── frontend/  # React + Vite (RTL, Persian UI)
```

- **بک‌اند**: Express روی SQLite (`better-sqlite3`) با احراز هویت JWT، نقش‌های `member`/`admin`، و مسیرهای REST برای تورنمنت‌ها، مسابقات، اخبار، آمار، کاربران و خبرنامه.
- **فرانت‌اند**: React 18 + React Router با فونت متغیر Vazirmatn (سبک‌ترین فونت فارسی با پوشش کامل وزن‌ها در یک فایل) و آیکون‌های سبک `lucide-react`.
- **پنل مدیریت**: مسیر `/admin` (فقط برای نقش admin) برای مدیریت کامل تورنمنت‌ها، مسابقات، اخبار و کاربران.

## اجرا در محیط توسعه

### ۱. بک‌اند

```bash
cd backend
npm install
cp .env.example .env   # در صورت نیاز مقادیر را تغییر دهید
npm run dev             # http://localhost:4000
```

اولین اجرا به‌صورت خودکار دیتابیس SQLite را در `backend/data/fifasoul.db` می‌سازد و با داده‌های نمونه seed می‌کند.

کاربر ادمین پیش‌فرض: `admin@fifasoul.test` / `admin123`

### ۲. فرانت‌اند

```bash
cd frontend
npm install
npm run dev              # http://localhost:5173
```

درخواست‌های `/api/*` در حالت توسعه به‌صورت خودکار به بک‌اند (پورت ۴۰۰۰) پروکسی می‌شوند.

## دیپلوی روی Railway (سرویس واحد)

بک‌اند در محیط production فرانت‌اند build‌شده را هم مستقیماً از پوشه `frontend/dist` سرو می‌کند؛ یعنی فقط یک سرویس روی Railway کافی است. تنظیمات این سرویس دقیقاً همانی است که یک بار در Railway پیکربندی می‌شود:

- **Root Directory**: `backend`
- **Build Command**: `npm install`
- **Start Command**: `npm start`

با این‌که Build Command فقط `npm install` است، فرانت‌اند هم به‌صورت خودکار ساخته می‌شود: یک اسکریپت `postinstall` در `backend/package.json` (فایل `backend/scripts/build-frontend.js`) بعد از نصب پکیج‌های بک‌اند اجرا می‌شود، وارد `../frontend` می‌شود، `npm install` و `npm run build` را همان‌جا اجرا می‌کند و خروجی را در `frontend/dist` می‌گذارد — دقیقاً جایی که بک‌اند انتظارش را دارد. این اسکریپت فقط وقتی روی Railway اجرا می‌شود کاری انجام می‌دهد (با تشخیص متغیرهای محیطی `RAILWAY_*`)؛ در اجرای معمولی `npm install` روی سیستم شما (خارج از Railway) بدون هیچ اثری رد می‌شود، بنابراین توسعه محلی کاملاً بدون تغییر می‌ماند.

### متغیرهای محیطی

| متغیر | مقدار |
|---|---|
| `JWT_SECRET` | یک رشته تصادفی و امن (شما قبلاً تنظیم کرده‌اید) |
| `PORT` | نیازی به تنظیم دستی نیست؛ Railway خودش مقداردهی می‌کند و بک‌اند از `process.env.PORT` استفاده می‌کند |
| `DATABASE_PATH` | اختیاری. اگر ست نشود، روی Railway به‌صورت خودکار به `<مسیر mount شده ولوم>/fifasoul.db` می‌رود (با تشخیص `RAILWAY_VOLUME_MOUNT_PATH`، یا `/data/fifasoul.db` اگر آن متغیر در دسترس نبود) و در حالت توسعه محلی به `backend/data/fifasoul.db`. برای اطمینان کامل می‌توانید صریحاً `DATABASE_PATH=/data/fifasoul.db` را هم در Railway ست کنید (چون Volume را روی `/data` mount کرده‌اید) |

### Volume برای پایداری داده‌ها

شما یک **Volume** روی مسیر `/data` به سرویس متصل کرده‌اید — همین کافی است. چون دیتابیس SQLite یک فایل روی دیسک است، بدون این Volume با هر ری‌استارت یا دیپلوی جدید داده‌ها از بین می‌رفتند. کد به‌صورت خودکار پوشه‌ی مسیر `DATABASE_PATH` را (در صورت نبودن) می‌سازد، پس نیازی به کار دستی دیگری نیست.

### بعد از دیپلوی

با اولین اجرا، اسکیمای دیتابیس و داده‌های نمونه به‌صورت خودکار روی همان Volume ساخته می‌شوند (دقیقاً مثل حالت توسعه، فقط با مسیر متفاوت). آدرس سرویس Railway هم API (`/api/*`) و هم فرانت‌اند را روی یک دامنه سرو می‌کند، بنابراین نیازی به تنظیم CORS یا آدرس API جداگانه در فرانت‌اند نیست.

## امکانات

- ثبت‌نام / ورود کاربران با رمزنگاری bcrypt و توکن JWT
- مدیریت تورنمنت‌ها (لیگ/کاپ) و ثبت‌نام کاربران در آن‌ها
- مدیریت مسابقات با وضعیت زنده/در انتظار/پایان‌یافته
- بخش اخبار با دسته‌بندی (بازی‌های فعال، جدیدترین‌ها، گیک‌ها، آموزش‌ها)
- آمار خانه (رو در رو، پلی‌آف، لیگ، کاپ) و جدول اعضای برتر — همگی از دیتابیس واقعی محاسبه می‌شوند
- عضویت در خبرنامه با شماره موبایل
- پنل مدیریت کامل با CRUD برای تورنمنت‌ها، مسابقات، اخبار و کاربران
